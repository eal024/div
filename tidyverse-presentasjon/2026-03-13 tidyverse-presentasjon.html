Hvorfor tidyverse?

Samme oppgave — to måter:

Base R 

df <- penguins[
  !is.na(penguins$bill_length_mm), ]

df$ratio <- df$bill_length_mm /
            df$bill_depth_mm

aggregate(ratio ~ species,
          data = df, FUN = mean)

tidyverse 

penguins |>
  filter(!is.na(bill_length_mm)) |>
  mutate(
    ratio = bill_length_mm /
            bill_depth_mm
  ) |>
  summarise(
    snitt = mean(ratio),
    .by   = species
  )

Uten pipe

Kode leses innenfra og ut 

summarise(
  mutate(
    filter(penguins,
           !is.na(bill_length_mm)),
    ratio = bill_length_mm /
            bill_depth_mm
  ),
  snitt = mean(ratio),
  .by = species
)

filter · select · mutate · summarise

filter() — velg rader

Behold rader som oppfyller en betingelse 

penguins |>
  filter(
    species == "Chinstrap",
    !is.na(body_mass_g)
  )

Flere betingelser separeres med komma — alle må være sanne (AND).

select() — velg kolonner

Behold utvalgte kolonner 

penguins |>
  select(species, island, body_mass_g)

Bruk -kolonne for å fjerne, starts_with() / ends_with() for mønstre.

mutate() — lag nye variabler

Legg til eller endre kolonner 

penguins |>
  mutate(
    masse_kg  = body_mass_g / 1000,
    stor_fugl = body_mass_g > 4500
  ) |>
  select(species, body_mass_g,
         masse_kg, stor_fugl)

summarise() — aggreger per gruppe

Reduser til sammendrag per gruppe 

penguins |>
  filter(!is.na(body_mass_g)) |>
  summarise(
    snitt_g = mean(body_mass_g),
    n       = n(),
    .by     = species
  )

pivot_longer() — bredt → langt

Samle flere kolonner til rader 

penguins |>
  select(species, bill_length_mm,
         bill_depth_mm) |>
  pivot_longer(
    cols      = ends_with("mm"),
    names_to  = "mål",
    values_to = "verdi_mm"
  )

Nyttig for plotting — ggplot2 liker langt format.

pivot_wider() — langt → bredt

Spre rader ut til kolonner 

penguins |>
  filter(!is.na(body_mass_g)) |>
  summarise(
    snitt = mean(body_mass_g),
    .by   = c(species, island)
  ) |>
  pivot_wider(
    names_from  = island,
    values_from = snitt
  )

Lesbar og reproduserbar kode
fem prinsipper

1. Prosjektstruktur

Aldri setwd() — bruk RStudio-prosjekt 

mitt-prosjekt/
├── data/
│   ├── raw/        # les kun, aldri skriv
│   └── processed/
├── scripts/
├── output/
│   ├── figures/
│   └── tables/
├── renv.lock
└── README.md

Bruk here::here() for alle stier 

# Fungerer på alle maskiner
read_csv(here::here(
  "data", "raw", "survey.csv"
))

# Brytes på andres PC
read_csv("C:/Users/eirik/
         prosjekt/data/survey.csv")

here finner .Rproj-filen og forankrer alle stier der — uavhengig av OS.

2. Kommenter hvorfor, ikke hva

Unngå dette 

# Filtrer rader
df |> filter(age > 18)

# Lag ny variabel
df |> mutate(ratio = a / b)

Gjør dette 

# Ekskluder mindreårige (jf. protokoll)
df |> filter(age > 18)

# Nebbdybde-ratio brukes i modell 2
df |> mutate(ratio = a / b)

Bruk seksjonsoverskrifter — vises i RStudios Outline-panel: 

## 1. Import og rensing ─────────────────────────
## 2. Konstruer variabler ────────────────────────
## 3. Modeller og figurer ─────────────────────────

3. Lesbar R-kode

Navngiving — snake_case, alltid 

# Variabler: substantiv
sick_leave_days <- 14
monthly_benefit <- 22000

# Funksjoner: verb
calculate_rate <- function(x, n) {
  x / n
}

Unngå camelCase, punktum (my.var), og enkeltbokstaver.

Pipe: én operasjon per linje 

# Vanskelig å lese
summarise(filter(df, !is.na(x)),
          snitt = mean(x))

# Lesbart
df |>
  filter(!is.na(x)) |>
  summarise(snitt = mean(x))

styler: Addins → Style active file — automatisk formatering.

4. RStudio — viktigste innstilling

Tools → Global Options → Workspace
Restore .RData on startup: Never
Save workspace on exit: Never
Tvinger deg til å skrive skript som alltid fungerer fra topp til bunn.
renv — lås pakkeversjonene
renv::init()
renv::snapshot()
renv::restore()
Lagrer renv.lock — en kollega kan reprodusere nøyaktig ditt miljø.

Kode som bare fungerer fordi du husker noe fra forrige økt er ikke reproduserbar — det er flaks.

5. Filorganisering

Nummererte skript — ett skript, én oppgave 

scripts/
├── 01_import.R      # kun innlasting
├── 02_clean.R       # rensing og koding
├── 03_analyse.R     # modeller
├── 04_visualise.R   # figurer
└── run_all.R        # kjør alt

run_all.R — én kommando kjører alt 

source(here::here(
  "scripts", "01_import.R"))
source(here::here(
  "scripts", "02_clean.R"))
source(here::here(
  "scripts", "03_analyse.R"))
source(here::here(
  "scripts", "04_visualise.R"))

Rådata er hellig — les kun, aldri overskriv. Lagre prosesserte data til data/processed/.
